# V2 质量提升方案

## 当前状况分析

### ✅ 已有能力
1. **回归测试框架**：5 个真实场景样例，全部可成功渲染
2. **基础质量检查**：`quality_checks.py` 已实现标题长度、内容密度等警告
3. **人工评审模板**：`review.md` 定义了 5 维度评分标准（1-5 分）
4. **评分计算工具**：`score_review.py` 可自动计算总分和评级
5. **测试覆盖**：113 个单元测试全部通过

### ❌ 核心缺口
1. **评审模板未填写**：5 个回归样例的 review.md 都是空的，没有基线数据
2. **质量闸门缺失**：warning 只记录不拦截，无法阻止低质量输出
3. **AI 生成不稳定**：
   - 标题可能过长（如 s5 超过 24 字符）
   - 内容密度难控制
   - 结构合理性无保障
4. **缺少自动化监控**：无法追踪质量趋势和退化

## 提升方案（按优先级）

### Phase 1: 建立质量基线（最高优先级）

**目标**：为 5 个回归样例建立人工评分基线

**行动**：
1. 打开每个生成的 PPT，按 5 维度评分
2. 填写 review.md 的评分和问题描述
3. 运行 `score_review.py --write` 自动计算总分
4. 建立当前质量水平的量化认知

**产出**：
- 5 个完整的 review.md（含评分、问题、建议）
- 质量基线报告（平均分、问题分布）

### Phase 2: 增强质量检查机制

**目标**：从 warning 升级为可配置的质量闸门

**行动**：
1. 扩展 `quality_checks.py`：
   - 增加结构合理性检查（首页/尾页类型、section 分布）
   - 增加内容质量检查（标题结论性、bullet 具体性）
   - 支持 error 级别（不仅是 warning）
2. 在 `ppt_engine.py` 中增加质量阈值配置
3. 当 error 数量超过阈值时拒绝生成

**产出**：
- 可配置的质量闸门
- 质量检查覆盖率提升到 10+ 项

### Phase 3: 优化 AI Prompt

**目标**：从源头提升生成质量

**行动**：
1. 在 `v2_outline.md` 中增强约束：
   - 明确标题长度限制（中文 ≤20 字）
   - 强调结论导向而非目录式标题
   - 要求首页设定场景、尾页收敛结论
2. 在 `v2_slides.md` 中增强约束：
   - 每个 bullet 长度限制（≤35 字）
   - bullet 数量建议（3-5 条）
   - 避免空泛表达的示例
3. 增加 few-shot 示例（好/坏对比）

**产出**：
- 优化后的 prompt 模板
- 回归测试验证改进效果

### Phase 4: 自动化质量监控

**目标**：持续追踪质量趋势

**行动**：
1. 扩展 `run_regression.py`：
   - 自动运行质量检查
   - 生成质量报告（warning/error 分布）
   - 与基线对比，检测退化
2. 增加 CI 集成（如果有）
3. 定期（每周/每次改动后）运行回归

**产出**：
- 自动化质量报告
- 质量趋势可视化
