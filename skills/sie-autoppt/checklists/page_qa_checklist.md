# Page QA Checklist (Required)

每一轮生成后必须执行此检查，至少完成一轮“修复 -> 复检”。

## A. 结构检查

- [ ] 本页是否只有一个主结论
- [ ] 信息层级是否清晰（标题/要点/注释）
- [ ] 是否存在纯文字大段堆叠
- [ ] 是否保留了 HTML 源页面的核心语义

## B. 视觉检查

- [ ] 元素是否重叠、遮挡、裁切
- [ ] 文本是否溢出、换行异常、边距过小
- [ ] 对齐与间距是否一致（卡片、标题、数据区）
- [ ] 对比度是否足够（投屏可读）
- [ ] 是否具备至少一个视觉锚点

## C. 业务表达检查

- [ ] 结论是否前置，听众 5 秒能抓住重点
- [ ] 关键数据是否可解释（指标 + 含义）
- [ ] 行动建议是否明确（若本页需要）

## D. 闭环要求

- [ ] 本轮至少记录 3 条可执行修复项
- [ ] 修复后已对受影响页面复检
- [ ] 无新增严重问题后才允许进入导出
