# HTML -> PPT Rebuild Rules

目标不是机械截图式复刻，而是“保留语义 + 提升演示表达”。

## 1) 双目标原则

每页都要同时满足：

- 复刻度：保留原 HTML 的信息架构、品牌语义、核心视觉关系
- 演示可读性：适配 PPT 的信息密度、讲述节奏和投屏可读性

建议默认权重：`复刻 70 / 优化 30`。若原页面信息过密，可切换为 `60 / 40`。

## 2) 元素映射（先映射，再绘制）

先把 HTML 模块映射为 PPT 构件：

- Hero 区 -> 封面页/章节页主视觉区
- 导航栏 -> 页眉弱化元素或删除
- 卡片栅格 -> 2x2 或 3x2 信息卡
- 数据表 -> 原生表格或摘要卡 + 详情附页
- CTA 区 -> 结尾行动页

禁止直接逐像素复刻整个网页截图。

## 3) 信息重构规则

1. 一页只保留一个主结论。
2. 每页最多 3 层信息层级（标题、要点、注释）。
3. 段落改短句，优先列表化与并列结构。
4. 关键数据做“数字卡”或对比卡，不埋在长文案里。

## 4) 视觉重构规则

1. 每页必须有视觉锚点（图、图标、图表、数据卡至少一项）。
2. 不做纯文字页（过渡页除外）。
3. 保留品牌色语义，但提升对比度和投屏可读性。
4. 同一 deck 内，封面/目录/结尾要同风格闭环。

## 5) 页级输出要求（执行时必须显式写出）

每页先输出四项再生成：

- 页面目标（一句话）
- 复刻来源（对应 HTML 哪个区域）
- PPT化改造点（2-4 条）
- 复刻/优化权重（70/30 或 60/40）

## 6) 禁止事项

- 禁止把网页完整截图塞进 PPT 当主内容
- 禁止把导航、按钮、表单控件原样照搬为无意义装饰
- 禁止为了“像”牺牲可读性（字号过小、对比不足、密度过高）
