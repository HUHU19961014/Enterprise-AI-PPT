## 一、项目概述

### 1.1 项目定位

Enterprise-AI-PPT 是面向企业汇报场景的 AI PPT 生成与交付工具，核心价值在于将 AI 智能规划与确定性 PPTX 渲染结合，输出可编辑、可复核、可追溯的企业级交付件。

### 1.2 技术栈概览

| 类别 | 技术选型 | 评估 |
|------|---------|------|
| Python 版本 | >=3.11 | ✅ 现代化选择 |
| 核心依赖 | python-pptx, openai, pydantic, beautifulsoup4 | ✅ 成熟稳定 |
| 数据验证 | Pydantic v2 | ✅ 现代化类型安全 |
| 测试框架 | pytest + coverage (目标 80%) | ✅ 有覆盖率要求 |
| 代码质量 | ruff + mypy | ✅ 静态检查配置完整 |

### 1.3 核心模块统计

| 模块 | 文件数 | 核心职责 |
|------|--------|----------|
| V2 语义流水线 | 24 个 Python 文件 | AI规划→语义Deck→编译→渲染 |
| 测试套件 | 69 个测试文件 | 覆盖 CLI、Schema、Render、Quality |
| 渲染器 | 10 种布局 | section_break, timeline, stats_dashboard 等 |

## 三、架构设计评估

### 3.1 整体架构评分

```
评分: ⭐⭐⭐⭐ (4/5)
```

**优点**:
- 分层清晰：CLI → Services → Semantic Compiler → Renderers
- V1/V2 路径隔离，legacy 模块有明确边界
- 依赖注入模式，便于测试

**待改进**:
- `deck_director.py` 混合了 schema shaping、normalization、routing 三种职责 (文档已承认)
- V2 renderers 存在布局常量重复逻辑

### 3.2 V2 流水线架构

```
输入 → Clarifier → AI Outline → AI Semantic Deck → Compiler → Quality Gate → Rewrite → SVG → PPTX → Review
```

| 阶段 | 模块 | 评分 | 说明 |
|------|------|------|------|
| 需求澄清 | clarifier*.py | ⭐⭐⭐⭐ | Web/CLI 双入口 |
| 智能规划 | services.py | ⭐⭐⭐⭐⭐ | 支持 deep/quick 双模式 |
| 语义编译 | semantic_compiler.py | ⭐⭐⭐⭐ | 本地确定性编译 |
| 质量门控 | quality_checks.py | ⭐⭐⭐⭐⭐ | 规则引擎 + 自动评分 |
| 内容重写 | content_rewriter.py | ⭐⭐⭐⭐ | 自动修复质量问题 |
| 视觉审查 | visual_review.py | ⭐⭐⭐⭐⭐ | 9维度评分 + blocker patches |
| 渲染输出 | ppt_engine.py + renderers/* | ⭐⭐⭐ | 存在已知 bug |

### 3.3 数据流架构

```
Topic/Brief → Structured Context → Strategy Analysis → Outline → Semantic Deck → Compiled Deck → PPTX
```

**优点**:
- 中间件可审计：outline.json, semantic_deck.json, deck.json 全链路保留
- Schema 驱动：Pydantic v2 提供强类型保障
