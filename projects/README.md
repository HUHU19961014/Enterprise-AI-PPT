# Projects Workspace

`projects/` is a local working area for generated decks, review batches, and ad-hoc demos.

## Intended Use

- `generated/`: legacy or manual local generation artifacts
- `visual_review/`: local visual QA batches
- other subfolders: temporary experiments or manually curated project workspaces

## Repository Policy

- Treat this directory as runtime workspace, not as the source-of-truth for bundled examples.
- Bundled sample inputs now live in `samples/input/`.
- Historical notes about old tracked demos belong in `docs/archive/`.
